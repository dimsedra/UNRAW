# Contributing to ATMOS // UNRAW

Thank you for your interest in contributing to the **ATMOS // UNRAW** ecosystem. We value contributions that align with our core design philosophy: *Deliberate, Not Decorative.*

## 🏛️ Design Philosophy

Before submitting a pull request, ensure your changes:
- Prioritize architectural minimalism.
- Maintain high-fidelity interactive physics.
- Adhere to the curated and professional-house aesthetic of the ATMOS brand.

## 🛠️ Development Workflow

1. **Fork the Repository**: Create your own branch for the feature or fix.
2. **Setup Locally**: Follow the installation steps in the [README](README.md).
3. **Draft Your Changes**: Ensure your code is clean, type-safe, and well-documented.
4. **Lint & Build**: Always run the validation suite before submitting:
   ```bash
   npm run lint
   npm run build
   ```

## 🌑 Pull Request Guidelines

- **Clarity**: Provide a detailed description of what your change accomplishes.
- **Atomic Commits**: Keep your commits focused and descriptive.
- **Visual Evidence**: If your change affects the UI, please include screenshots or screen recordings.
